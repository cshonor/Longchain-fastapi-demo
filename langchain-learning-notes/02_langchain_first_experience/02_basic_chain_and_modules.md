## LangChain 基础链核心要素与五大模块关系总结

本笔记用于把「**基础链三要素**」与「**常见五大模块**」的关系一次讲清楚，便于建立整体框架感。

---

### 一、基础链的三大核心要素

LangChain 中最基础、最核心的执行单元可理解为**基础链（Basic Chain）**，由三个必不可少的组件构成，是所有复杂应用的底层基础：

1. **提示词模板（Prompt Template）**  
   负责构建标准化、可复用的输入指令，支持动态变量填充，统一向大模型传递任务要求与格式约束。

2. **语言模型（LLM / ChatModel）**  
   整个系统的“大脑”，负责理解输入、执行推理、生成文本内容，是 AI 能力的核心来源。

3. **输出解析器（Output Parser）**  
   将模型生成的自然语言文本，转换为代码可直接处理的结构化结果（字符串、JSON、列表等），方便后续业务逻辑使用。

这三者常通过 LCEL 风格的「可组合表达」串联，形成一条可运行的基础链：

```
提示词模板 → 语言模型 → 输出解析器
```

---

### 二、三大核心要素与五大模块的关系

从学习与理解角度，LangChain 常被划分为五类能力：**模型 I/O（IO）模块、RAG 模块、功能模块（Agent/Tools）、存储模块、回调模块**。  
它们与三大核心要素的关系可以概括为：

**三大核心是底层最小执行单元；五大模块是在这些单元之上组织出来的功能体系。**

#### 1. IO 模块（模型 I/O）

直接对应并**完全包含三大核心要素**：

- 输入：Prompt Template  
- 处理：LLM  
- 输出：Output Parser  

可以理解为：**三大核心 = IO 模块的主体内容**。

#### 2. RAG 模块（检索增强生成）

在 IO 模块基础上增加「检索与上下文注入」能力：

```
检索文档 → 塞入 Prompt → LLM 生成 → Parser 解析
```

要点：三大核心依然是最终生成回答的执行部分；RAG 的关键差异在于**让 Prompt 拿到外部资料**。

#### 3. 功能模块（Agent / 工具链）

用于构建智能体、多步决策、工具调用等复杂逻辑。  
无论逻辑多复杂，**最终的“生成与表达”仍会落到三大核心**（Prompt/LLM/Parser）上执行。

#### 4. 存储模块

用于保存与读取应用状态数据，例如：

- 对话历史 / 记忆（Memory）
- 缓存（Cache）
- 运行中间结果或业务状态

它主要为 Prompt 提供**可复用的上下文来源**。  
说明：向量库（Vector Store）常在 RAG 流程里使用，概念上更贴近「检索与索引」；但工程落地时也经常归入“存储/基础设施”层一起管理。

#### 5. 回调模块（Callback / 日志监控）

不直接参与生成逻辑，用于追踪、日志、调试、性能统计与可观测性。  
它会在 IO/RAG/Tools 等执行过程中，监听并记录关键阶段的输入输出与耗时等信息。

---

### 三、整体关系总结

- **三大核心要素**：LangChain 的**最小执行单元**，相当于“发动机”。  
- **五大模块**：基于核心组件封装的**功能层**，相当于“整车系统”。  
- 各模块最终都会回到三大核心完成「输入组织 → 生成 → 解析」这件事；它们的差异主要体现在：**Prompt 的材料从哪里来、流程如何编排、如何调用外部能力、如何记录与观测**。
